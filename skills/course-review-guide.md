# 课程复习文档生成器（HTML + PDF 双格式）

## 概述

将课程原始课件一键转化为高质量、结构化的**单文件HTML交互式考试复习文档**和**PDF打印版复习文档**。输出包含MathJax数学排版、内联SVG示意图、课件原图（严格过滤）、完整公式推导、例题详解（含举一反三变体题）、交互式学习组件、答题模板和常见错误附录。

**关键改进（v2）**：
- 📐 理科课件：图片-公式强制对应，无用图片自动过滤
- 📖 文科课件：概念辨析、时间线、论述框架独立处理
- ✏️ 例题深度解析：考点分析→策略选择→步骤讲解→验证→易错点→举一反三
- 📄 双格式输出：HTML + PDF 同时生成，PDF中文不乱码

## 子技能索引

本技能按需引用以下子技能。当遇到特定场景时，自动加载对应规则：

| 子技能文件 | 触发场景 | 核心内容 |
|-----------|---------|---------|
| `extraction-rules-science.md` | 课件含公式/代码/计算/推导 | 图片-公式对应、无用图过滤、推导补全、计算题处理 |
| `extraction-rules-humanities.md` | 课件含概念论述/时间线/案例分析 | 概念辨析表、时间线、论述框架、记忆口诀 |
| `example-handling.md` | 任何时候——必须提取课件例题 | 例题识别、解题思路讲解、举一反三变体题 |
| `output-generation.md` | 生成最终文档时 | HTML+PDF双格式、中文编码、字体配置 |

## Phase 1：信息收集与范围确认

### 交互式模式（Claude Code / Codex / OpenCode）

**A1. 确认课件位置**：询问课件目录路径。

**A2. 扫描课件文件**：列出所有 `.pdf`/`.pptx`/`.docx` 文件，呈现文件名和大小。

**A3. 确认考试范围（🔴 必须逐项询问）**：
1. 考试范围：哪些章节/讲座在考试范围内？哪些不考？
2. 课程全称（中/英文）
3. 授课教师（如用户不愿透露则跳过）
4. 参考教材
5. 考试形式：闭卷/开卷/半开卷
6. 授课语言
7. 输出文件名
8. 是否需要押题文档

**A4. 创建输出目录**：`extracted_text/` 和 `extracted_images/`。

### 自主模式

跳过询问，从 `exam-scope.json` 或文件名推断范围，生成时插入自主模式横幅。

## Phase 2：课件内容提取

### 2.1 运行提取脚本

```bash
python extract_course_materials.py --course-dir "课件目录" --output-dir "输出目录" [--render-pages]
```

多模态模型加 `--render-pages`，非多模态模型不加。

### 2.2 读取提取结果并分类

读取所有 `extracted_text/*.txt`。**在读取过程中自动判定每章属于理科/文科/混合类型**：

- 含公式+计算+代码 → 启用 `extraction-rules-science.md`
- 含概念论述+时间线+案例（无公式） → 启用 `extraction-rules-humanities.md`
- 两者都有 → 两者并存，按章节分别处理

### 2.3 🔴 例题识别与提取（必须在读取阶段完成）

在读取提取文本时，**立即执行 `example-handling.md` 第1节的例题识别规则**。所有被识别的例题必须：

1. 标记在对应章节中
2. 记录题目原文、所有选项/条件
3. 如果题目的解答在课件中不完整，标注 `<!-- 待补全 -->`

## Phase 3：补充调研（可选）

搜索 GitHub/Wikipedia 补充理解。原则：
- 仅辅助理解，不替代课件
- 外部内容与课件冲突时以课件为准
- 不确定处标注 `<!-- 待确认 -->`

## Phase 4：生成HTML复习文档

### 4.0 生成方法

采用串行Python追加模式（禁止使用子Agent）：
1. Write工具创建HTML头部+CSS+导航结构
2. Python脚本逐批追加2-3章内容
3. 最后追加附录+页脚+JavaScript

### 4.1 生成前必须完成的思考清单

1. 课程核心主线是什么？
2. 哪些概念是"基石"？
3. 哪些公式/定理必考？
4. 学生最容易在哪些地方混淆？
5. **课件中每道例题是什么？考点是什么？解法关键步骤是什么？**（🔴 新增）
6. **每道例题的举一反三变体题是什么？**（🔴 新增）
7. 哪些图片是真正有用的（对应公式/概念）？哪些是无用的（logo/装饰/重复）？
8. 每章内容属于理科处理还是文科处理？

### 4.2 内容生成规则

生成每章内容时，**按需加载子技能规则**：

**理科章节**（启用 `extraction-rules-science.md`）：
- 图片经过三级过滤后才引用
- 每张引用的图片必须有对应公式/概念
- 推导补全，每步有"为什么"
- 公式完整性逐一检查

**文科章节**（启用 `extraction-rules-humanities.md`）：
- 概念辨析表格
- 时间线整理
- 记忆口诀

**所有章节**（强制启用 `example-handling.md`）：
- 🔴 每道例题按完整解析模板展开
- 🔴 每道例题附"举一反三"变体题
- 例题索引表

**输出格式**（强制启用 `output-generation.md`）：
- HTML + PDF 双格式
- PDF中文不乱码的CSS配置

### 4.3 文档结构（8个部分）

```
1. 封面信息（frontmatter）
2. 📖 阅读指南
3. 📑 目录
4. 第〇章：开始之前—课程核心思维
5. 各章节精讲（含例题+举一反三）
6. 附录A：公式速查卡
7. 附录B：解题模板
8. 附录C：常见错误与陷阱
```

## Phase 5：双格式输出（🔴 HTML + PDF）

按 `output-generation.md` 执行：

```bash
# Step 1: 生成HTML（已完成）
# Step 2: 内嵌图片
python embed_images.py <文件名>.html --in-place

# Step 3: 生成PDF
python -m weasyprint <文件名>.html <文件名>.pdf

# Step 4: 清理临时脚本
```

如weasyprint不可用，提示用户通过浏览器打印为PDF。

## Phase 6：质量检查

### HTML检查
- [ ] 浏览器正常渲染，MathJax公式正确
- [ ] 所有公式 `$$` 成对出现
- [ ] 图片引用全部有效（路径或base64）
- [ ] 每道例题有考点分析+解题策略+步骤讲解+举一反三
- [ ] 每张图有对应的公式/概念说明
- [ ] 搜索/闪卡/选项卡/进度追踪功能正常
- [ ] 无logo、装饰、重复等无用图片
- [ ] 移动端和打印样式正常
- [ ] 页脚不含作者姓名

### PDF检查
- [ ] 中文无乱码/方块
- [ ] 公式正常渲染
- [ ] 图片可见且位置正确
- [ ] 分页合理
