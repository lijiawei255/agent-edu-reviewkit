# Course Note — 课件逐章笔记生成器

## 核心定位

本技能不再以“考前复习总结”为默认目标，而是将课程原始课件整理为可长期学习、查阅和二次编辑的章节笔记。

默认规则：

- **一个章节 = 一个章节笔记**。扫描课件时必须先识别章节编号/章节名；如果多个 PDF/PPTX/DOCX 属于同一章，必须合并整理成同一个笔记。
- **一个课件文件不一定等于一个章节**。文件名带有 `Part1/Part2`、`[1-2]/[3-4]`、`上/下`、`第1讲/第2讲` 等分片标记时，优先判断它们是否属于同一章。
- **每章输出一个独立文件**，文件名为：`章节名+笔记.md` 或 `章节名+笔记.html`。章节名应尽量使用稳定的归并名，而不是某个分片课件名。
- **输出格式必须询问用户后决定**：Markdown 或 HTML。
- **主内容来自课件**，外部资料只能用于辅助解释、补全背景或校验术语，不能替代课件内容。
- **笔记目标是“学懂这一章”**，不是只列考点；需要保留概念脉络、公式、图表、推导、例题和易混点。
- **根据模型能力选择路径**：多模态模型优先看整页截图/PDF 页面；非多模态模型使用提取文本和抽出的图片素材。

## Phase 1: 范围扫描与必要确认

### 1.1 确认课件目录

如果用户已经给出目录，直接扫描该目录；否则询问用户课件所在目录。

支持文件：

- `.pdf`
- `.pptx` / `.pptm`
- `.docx` / `.dotx` / `.dotm`

### 1.2 扫描课件

列出所有支持的课件文件，至少包含：

- 文件名
- 文件格式
- 文件大小
- 自动识别的章节键
- 自动推断的章节组名
- 自动推断的笔记名

### 1.3 章节检测与同章归并

扫描后必须先建立“课件文件 -> 章节组”的映射，再开始生成笔记。不要直接按文件数量生成笔记。

章节键识别优先级：

1. **文件名显式章节号**：识别 `Chapter1`、`Chapter 1`、`Ch1`、`CH01`、`第1章`、`第01章`、`Chap.7` 等模式，统一为 `Chapter{N}`。
2. **文件名分片标记**：若文件名还包含 `Part1/Part2`、`[1-2]`、`[3-4]`、`Week`、`Lecture`、`上/下` 等，只作为同章内部顺序，不作为新的章节键。
3. **课件首页/目录页标题**：当文件名不含章节号时，读取提取文本前几页或整页截图，寻找 `Chapter N`、`第N章`、章节标题。
4. **主题标题相似度**：章节号缺失但标题高度相似时，可归为同一章节；不确定时保留为独立组，并在报告中标注“章节归并待确认”。
5. **用户显式说明优先**：如果用户指出某几个课件属于同一章，必须以用户说明为准。

同章归并规则：

- 同一章节组内的课件按自然顺序排列：章节号 -> 分片号/页码范围 -> 文件名。
- 合并笔记时要去重课程介绍、重复公式、重复封面和目录页，只保留一次核心概念。
- 合并后的“课件来源”必须列出该章包含的所有源文件。
- 文件名使用章节组名，例如：

```text
{章节组名}+笔记.{md|html}
```

示例：

```text
Chapter1_Tension,+compression+&+Shear[1-2].pdf
Chapter1_Tension,+compression+&+Shear[3-4].pdf
Chapter1_Tension,+compression+&+Shear[5-7].pdf
  -> Chapter1_Tension,+compression+&+Shear+笔记.md

Chapter7_Analysis+of+stress+and+strain_Part1.pdf
Chapter7_Analysis+of+stress+and+strain_Part2.pdf
  -> Chapter7_Analysis+of+stress+and+strain+笔记.html

智能机电系统第2章.pdf
  -> 智能机电系统第2章+笔记.html
```

### 1.4 只询问必要信息

除非用户已经说明，否则只必须确认：

1. **输出格式**：Markdown 还是 HTML？
2. **输出位置**：如果用户未指定，默认在课件目录或当前项目输出目录下创建 `course_notes_output/`。

以下信息不再默认询问：

- 考试范围
- 考试形式
- 复习重点
- 是否汇总成总文档

只有当用户主动要求“面向考试”“只整理某几章”“汇总成一个文件”时，才进入对应的定制流程。

## Phase 2: 选择理解路径与生成素材

### 2.1 先判断模型能力

根据当前运行环境选择路径：

| 路径 | 适用模型/环境 | 核心素材 | 使用原则 |
|---|---|---|---|
| **A. 多模态优先路径** | GPT、Claude、Gemini 等可看图/页面的模型 | `extracted_text/` + `page_images/` | 用文本快速定位，用整页截图理解版式、图文关系、框图、曲线和公式 |
| **B. 非多模态降级路径** | DeepSeek 等不能直接看图的模型 | `extracted_text/` + `extracted_images/` | 依赖文本和图片文件名/位置标记，必要时让用户提供 OCR 或图片说明 |

不要把“逐张抽图并识别”作为多模态模型的默认前置步骤。对多模态模型，整页截图通常比抽出的内嵌图片更可靠，因为课件中的图表可能由矢量形状、文本框、箭头和多张小图组合而成。

### 2.2 使用项目提取脚本

优先使用项目根目录的 `extract_course_materials.py` 生成素材。

脚本能力：

- 提取 PDF/PPTX/DOCX 文本到 `extracted_text/`
- 提取内嵌图片到 `extracted_images/`，作为笔记引用素材和非多模态降级通道
- 可选将 PDF 每页渲染为整页截图到 `page_images/`，用于多模态模型理解页面整体
- 在文本中插入 `[图片: xxx.png]` 标记，记录图片位置
- 自动检测扫描版 PDF 或纯图片型文件
- 输出每个课件的页数/幻灯片数、图片数量和页面截图数量

推荐命令：

```bash
# 多模态模型推荐：生成文本、抽图和整页截图
python extract_course_materials.py --course-dir "课件目录" --output-dir "输出目录" --render-pages

# 非多模态模型或只需轻量提取：生成文本和抽图
python extract_course_materials.py --course-dir "课件目录" --output-dir "输出目录"
```

### 2.3 输出目录约定

建议目录结构：

```text
course_notes_output/
├── extracted_text/
├── extracted_images/
├── page_images/
├── 1-pdf+笔记.md
├── 2-pdf+笔记.md
└── ...
```

若输出为 HTML，同级文件改为 `.html`。

图片路径必须相对笔记文件可访问：

```markdown
![图示说明](extracted_images/xxx.png)
```

```html
<img src="extracted_images/xxx.png" alt="图示说明">
```

页面截图只用于理解课件，不默认直接嵌入笔记；若要引用整页截图，应明确说明“整页课件截图”，并避免让笔记变成截图合集。

### 2.4 多模态路径：文本定位 + 整页理解 + 素材引用

当模型可以看图或页面时：

1. 先读 `extracted_text/{课件文件名}.txt`，建立页码、章节、小节和关键词索引。
2. 对公式密集页、图表密集页、文本提取明显破碎页，查看 `page_images/{课件名}_p{页码}.png`。
3. 用整页截图理解：
   - 图和文字之间的关系
   - 框图、箭头、流程线和层级结构
   - 曲线坐标轴、图例和标注
   - 公式的真实排版和上下标
4. 从 `extracted_images/` 中选择少量真正适合引用的原图；如果抽图碎裂、缺箭头、缺文字或只是 logo，则不要引用，改用内联 SVG 重画概念图。
5. 生成笔记时优先写“图表解释”，而不是堆图片。

### 2.5 非多模态路径：文本 + 图片标记降级

当模型不能看页面或图片时：

1. 逐一读取 `extracted_text/`。
2. 用 `[图片: xxx.png]` 标记推断图片所在页和上下文。
3. 只引用上下文明确的图片；对无法确认的图片，标注“图像内容待确认”。
4. 对扫描版、纯图片型 PDF 或公式严重损坏页，要求用户提供 OCR 文本、截图说明或切换到多模态模型。

### 2.6 读取提取结果

每个章节组单独处理：

1. 读取该章节组内所有源文件对应的 `extracted_text/{课件文件名}.txt`
2. 记录每个源文件的页码/幻灯片边界和分片顺序
3. 多模态路径下查看该章节组的关键 `page_images/`
4. 建立图片标记、整页截图与源文件/页码的关系
5. 归并本章标题、主线、关键小节、公式和图表
6. 去除不同分片之间重复的封面、目录、定义和公式
7. 生成该章节组对应的独立笔记

对于内容较长的章节组，必须按源文件和页码分段读取，避免遗漏后半部分。

### 2.7 扫描版或纯图片课件

如果提取脚本报告某个文件为纯图片型课件：

- 多模态路径：直接读取整页截图，不依赖文本提取
- 如果当前环境没有可用视觉工具，在笔记中标记“需要 OCR/视觉补充”，不要编造内容
- 可以建议用户提供 OCR 文本或允许使用本地 OCR 工具

## Phase 3: 每章笔记生成规范

### 3.1 生成前思考

对每个章节组先回答：

1. 这一章的核心问题是什么？
2. 这一章有哪些基本概念？
3. 哪些公式、定理或流程是理解后续内容的基础？
4. 哪些图表真正承载了信息？
5. 课件中哪些推导、例题或步骤被省略，需要补全？
6. 学生最容易把哪些概念混淆？

### 3.2 每章笔记结构

每个“章节名+笔记”至少包含：

1. **封面信息**
   - 课程/章节名（能从课件推断则填写）
   - 课件来源文件名；若同章由多个课件合并，必须逐个列出
   - 生成日期
2. **本章导读**
   - 一句话主线
   - 学习目标
   - 推荐阅读顺序
3. **知识结构图**
   - 用列表或内联 SVG 展示本章概念关系
4. **分节笔记**
   - 概念定义
   - 关键公式
   - 图表解释
   - 推导补全
   - 例题/案例
   - 易混点
5. **本章小结**
   - 核心概念
   - 核心公式
   - 常见错误
6. **附录**
   - 术语表
   - 公式速查
   - 待确认内容（如有）

### 3.3 内容写作标准

- 关键术语优先给出中文，必要时附英文。
- 所有公式使用标准 LaTeX。
- 重要结论使用 `\boxed{}`。
- 公式不要只贴结果，要解释符号含义和适用条件。
- 课件里跳步的地方要补推导，但不能凭空添加课件完全未涉及的大段内容。
- 对易混概念使用对比表。
- 对抽象概念给出直观解释。
- 图片必须有说明文字，不能只插图。

## Phase 4: 页面、图片与公式策略

### 4.1 整页截图使用

整页截图的主要用途是**理解**，不是默认插入笔记。

优先查看整页截图的情况：

- 文本提取只剩碎片，无法判断图文关系
- 公式上下标、矩阵、分段函数等提取错误
- 课件页包含框图、流程图、控制结构图、坐标曲线
- 抽出的内嵌图片缺失文字、箭头、标注或被拆碎
- 扫描版/图片型 PDF

只有当整页本身是有价值的总览页、流程页或图表页时，才在笔记中引用整页截图，并说明它是“课件整页截图”。

### 4.2 抽出图片使用

`extracted_images/` 是素材池，不是必须全部识别或全部引用。

引用前必须过滤：

- logo
- 页眉页脚模板
- 纯装饰图
- Q&A/感谢页装饰图
- 与文字完全重复且无额外信息的公式截图

图片选择优先级：

1. 系统框图、结构图、流程图
2. 物理模型、几何示意图
3. 关键公式推导图
4. 对比图、实验结果图、曲线图
5. 典型例题图

每章建议至少引用 2-3 张有效图片；如果课件本身有效图片少，可以用内联 SVG 补充概念关系图。

### 4.3 图片-文本映射

必须利用提取文本中的 `[图片: xxx.png]` 标记来判断图片所在页码和上下文。

不要仅凭图片文件名猜测图片含义；无法确认时，宁可不引用，或在说明中标注“图像内容待确认”。

多模态路径下，如果抽图内容和整页截图上下文冲突，以整页截图为准。

### 4.4 公式还原

常见提取问题：

| 问题 | 修复方式 |
|---|---|
| 多行公式被拆散 | 根据上下文合并为完整 LaTeX |
| 上下标丢失 | 根据符号含义恢复为 `x_i`、`x^2` 等 |
| 分数线丢失 | 改写为 `\frac{a}{b}` |
| 希腊字母乱码 | 根据上下文恢复为 `\alpha`、`\omega`、`\theta` 等 |
| 函数名未规范 | 使用 `\sin`、`\cos`、`\lim`、`\exp` |

不确定的公式用 `<!-- 待确认：公式可能由提取错误导致 -->` 标注。

## Phase 5: 输出格式

### 5A: Markdown 输出

文件名：

```text
{章节名}+笔记.md
```

模板：

```markdown
# {章节名} 笔记

> **课件来源**: {原始文件名；多个源文件逐个列出}
> **生成日期**: {YYYY-MM-DD}

## 本章导读

- **主线**: ...
- **学习目标**: ...
- **阅读顺序**: ...

## 知识结构

...

## 1. 小节标题

### 📌 核心概念

...

### 🔑 关键关系/公式

$$
\boxed{...}
$$

### 🖼️ 图表解释

![图示说明](extracted_images/xxx.png)

### 📐 推导补全

...

### ✏️ 例题/案例

...

### ⚠️ 易混点

| 易混点 | 正确理解 |
|---|---|
| ... | ... |

## 本章小结

## 附录：术语与公式速查
```

Markdown 数学格式：

- 内联公式：`$...$`
- 显示公式：`$$` 单独成行
- 中文与 `$` 之间保留空格

### 5B: HTML 输出

文件名：

```text
{章节名}+笔记.html
```

HTML 必须：

- 单文件自包含 CSS
- 使用 MathJax 3 CDN 渲染公式
- 图片使用相对路径 `extracted_images/xxx.png`
- 支持移动端阅读
- 支持打印
- 有目录锚点
- 对关键概念、公式、推导、例题、易错点使用统一视觉样式

基本结构：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>{章节名} 笔记</title>
<script>
window.MathJax = {
  tex: {
    inlineMath: [['$', '$'], ['\\(', '\\)']],
    displayMath: [['$$', '$$'], ['\\[', '\\]']]
  },
  svg: { fontCache: 'global' }
};
</script>
<script async src="https://cdn.jsdelivr.net/npm/mathjax@3/es5/tex-svg.js"></script>
<style>
/* 使用清晰、克制、适合学习材料的排版；需包含响应式和打印样式 */
</style>
</head>
<body>
<!-- 生成完整章节笔记 -->
</body>
</html>
```

## Phase 6: 质量检查

每个输出文件生成后自检：

- [ ] 文件名符合 `章节名+笔记`。
- [ ] 已完成章节检测；同一章的多个课件已合并成同一个章节笔记。
- [ ] 每个章节组都生成了独立笔记。
- [ ] 合并章节的来源课件已完整列出，且分片内容没有遗漏。
- [ ] 笔记不是考试提纲，而是完整章节学习笔记。
- [ ] 课件来源已标注。
- [ ] 本章主线和学习目标清楚。
- [ ] 关键概念、公式、图表、推导、例题、易混点均有覆盖。
- [ ] 公式使用 LaTeX，重要结果使用 `\boxed{}`。
- [ ] 多模态路径下已查看公式/图表密集页的整页截图。
- [ ] 非多模态路径下已明确无法视觉确认的内容。
- [ ] 图片路径相对笔记文件可访问。
- [ ] 引用图片与整页上下文或文本上下文匹配。
- [ ] 对提取不确定的内容做了明确标注。
- [ ] Markdown/HTML 格式符合用户选择。
- [ ] HTML 输出可在浏览器中打开，MathJax 引用正确。

## 默认执行策略

当用户给出课件目录并要求生成笔记时：

1. 扫描目录。
2. 提取文件名、首页标题和目录信息，建立章节组。
3. 如果发现同一章的多个课件，先合并为同一章节组。
4. 判断当前模型是否具备多模态能力。
5. 多模态模型运行提取时优先加 `--render-pages`；非多模态模型可只提取文本和图片。
6. 询问输出格式（Markdown / HTML）。
7. 为每个章节组生成一个 `章节名+笔记` 文件。
8. 报告章节组映射、生成文件列表、提取文本数量、页面截图数量、引用图片数量和待确认事项。
