# 理科课件提取规则

## 适用范围

本规则适用于以下理科/工科课程类型：
- **数学类**：微积分、线性代数、概率论、复变函数
- **物理类**：力学、电磁学、光学、热力学
- **电路/电子类**：电路分析、模电、数电、信号与系统
- **计算机类**：数据结构、算法、操作系统、计算机网络、机器学习
- **工程类**：机械、控制、土木、化工
- **其他理工科**：包含公式、计算、推导、代码的课程

## 1. 图片过滤规则（🔴 核心：拒绝无用图片）

课件提取的所有图片必须经过以下三级过滤，**丢弃不满足条件的图片**。

### 1.1 必须丢弃的图片类型

以下类型的图片**一律不引用、不嵌入**到最终文档中：

| 类型 | 判定特征 | 处理 |
|------|---------|------|
| Logo/标识图 | 文件小（<5KB），出现在每页相同位置 | ❌ 丢弃 |
| 页眉页脚模板 | 多个文件出现相同尺寸/相似内容的图片 | ❌ 丢弃 |
| 纯装饰图 | 与教学内容无关联的风景、人物、抽象背景 | ❌ 丢弃 |
| 感谢/结束页装饰 | 出现在最后一页，含"谢谢"/"Thank you" | ❌ 丢弃 |
| 纯文字公式截图 | 文本提取中已有对应内容，无额外信息的公式图片 | ❌ 丢弃 |
| 重复图片 | 多个课件中出现完全相同的图片 | ❌ 仅保留首次出现 |
| EMF/WMF格式 | 浏览器不兼容（提取脚本已自动过滤） | ❌ 自动跳过 |
| 分辨率过低 | 宽或高<100px，无法辨认内容 | ❌ 丢弃 |

### 1.2 必须保留的图片类型（按优先级）

| 优先级 | 类型 | 判定特征 | 用途 |
|--------|------|---------|------|
| 🔴 P0 | 公式图解 | 图片附近有公式，图片展示公式的几何/物理含义 | 与公式配对展示 |
| 🔴 P0 | 算法流程图 | 含箭头、方框、判断菱形的步骤图 | 解题模板可视化 |
| 🔴 P0 | 数据结构/系统架构图 | 含节点和连接线的结构图 | 概念关系展示 |
| 🟡 P1 | 坐标曲线/函数图 | 含坐标轴、网格、曲线的图表 | 函数行为可视化 |
| 🟡 P1 | 对比示意图 | 含左右/上下两组对比的图 | 概念辨析 |
| 🟡 P1 | 实物/场景图 | 展示实际应用场景或实验装置 | 建立直觉 |
| 🟢 P2 | 例题配图 | 题目中引用的数据图表 | 辅助解题 |

### 1.3 图片-公式/概念对应规则（🔴 强制）

**每张被引用的图片必须有明确的公式或概念与之对应**：

1. 图片必须放在对应的公式/概念**之后、下一概念之前**
2. 图片下方必须有图注，格式：`图N.M：[该图说明的概念或公式]（来源：[课件文件名]）`
3. 图注不能只描述图片本身（如"一张包含坐标轴的图"），必须说明**图片展示的数学/物理含义**
4. 如果一张图片无法与任何公式或概念配对 → **丢弃该图片**

```
正确示例：
[傅里叶变换公式]
[图：展示矩形脉冲的时域和频域对应关系]
[文字：如上图所示，时域压缩对应频域扩展...]

错误示例：
[傅里叶变换公式]
[图：一个不相关的流程图]
[文字：傅里叶变换很重要...]
```

## 2. 公式提取与还原规则

### 2.1 公式完整性检查清单

提取每个公式后，逐项检查：

| 检查项 | 常见错误 | 修复方法 |
|--------|---------|---------|
| 上下标 | `x2` → `x^2` 或 `x_2` | 根据公式含义判断 |
| 分数线 | `a/b` → `\frac{a}{b}` | 分子分母有2个以上符号时必须用frac |
| 希腊字母 | α→`\alpha`, ω→`\omega`, σ→`\sigma` | 查希腊字母表还原 |
| 求和/积分 | ∑→`\sum`, ∫→`\int` | 补全上下限 |
| 矩阵 | 元素排列错乱 | 用`bmatrix`或`pmatrix`环境重新排列 |
| 分段函数 | 条件丢失 | 用`\begin{cases}`补全 |
| 根号 | √a→`\sqrt{a}` | 被开方数>1个字符时用`\sqrt[a]{b}` |
| 函数名 | sin→`\sin`, log→`\log` | 标准函数名用反斜杠 |

### 2.2 公式排版规范

- 独立公式：`$$` **必须单独成行**，公式内容在中间行
- 行内公式：使用单个 `$...$`，中文与 `$` 之间保留空格
- 重要结论使用 `\boxed{公式}`
- 多行推导使用 `\begin{aligned}` 等环境，在 `=` 处对齐
- 公式编号使用 `\tag{编号}`

## 3. 推导补全规则

### 3.1 补全优先级

| 优先级 | 情况 | 处理 |
|--------|------|------|
| 🔴 必须补 | 课件写"易证得""显然""由此可得"但无中间步骤 | 补全至少3步推导 |
| 🔴 必须补 | 核心定理/公式的推导 | 从定义出发，每步有解释 |
| 🟡 建议补 | 例题中的计算步骤跳跃 | 补全代数化简过程 |
| 🟢 可选补 | 纯代数化简的每一步 | 关键步保留，纯计算可省略 |
| ⚪ 不补 | 教材已有完整推导 | 引用教材页码即可 |

### 3.2 推导格式

每个推导步骤必须包含：
1. **做了什么操作**（如"将公式(1)代入公式(2)"）
2. **为什么做这一步**（基于什么原理/定理/技巧）
3. **完整的中间表达式**（LaTeX格式，不允许跳步）

使用可折叠的 `<details class="derive-steps">` 包裹完整推导。

## 4. 计算题处理规则（🔴 核心新增）

### 4.1 识别课件中的例题

扫描课件文本时，通过以下特征识别例题：

- 出现"例""例题""Example""习题""练习"等标记
- 出现问号结尾的题目描述，后跟解答步骤
- 代码块中有 `# 示例` 或类似注释
- 选项列表（A/B/C/D）配合"提交"按钮（课堂互动题）
- 课件中明确标注"考点""重点"的计算过程

### 4.2 例题提取模板

每道被识别的例题，按以下结构提取：

```
题目编号：对应课件中的题号
题目描述：完整抄录题目，包括所有已知条件
题目类型：概念辨析 / 计算题 / 编程题 / 证明题
所属知识点：该题考察的核心概念
解题步骤（逐步骤）：
  步骤1：[操作] → [为什么] → [中间结果]
  步骤2：[操作] → [为什么] → [中间结果]
  ...
最终答案：\boxed{答案}
易错点：做这道题最容易犯的2-3个错误
```

### 4.3 解题思路讲解规则（🔴 强制）

**禁止只列步骤不加解释。** 每道例题的每个解题步骤必须包含：

1. **考点分析**：本题考察的核心知识点、公式
2. **策略选择**：为什么选择这种方法？有没有其他方法？为什么不用？
3. **步骤解释**：
   - 步骤名称（如"计算经验熵"而非"步骤1"）
   - 这一步做了什么（具体计算）
   - 为什么做这一步（原理依据）
   - 这一步的中间结果
4. **验证方法**：如何检查答案是否正确（量纲检查、边界条件、代入验证等）

### 4.4 举一反三（🔴 每道例题必须生成变体题）

每道例题提取后，必须生成 **1-2道变体练习题**：

**变体题生成规则**：
- **同概念，不同数据**：保持核心概念完全相同，只改变数值/条件
- **同方法，不同场景**：保持解题方法不变，应用到新的上下文
- **不能引入新概念**：变体题必须只用例题已涉及的知识点
- **难度不高于原题**：变体题是巩固练习，不是拔高题

**变体题格式**：
```
📝 举一反三：[编号]
题目：[与例题同概念、同方法、不同数据的新题]
💡 提示：[简要提示解题方向，不完全公布答案]
```

## 5. 代码处理规则（计算机类课程）

### 5.1 代码识别

- 识别 `import`、`def`、`class`、`for`、`if` 等Python关键字
- 识别缩进块
- 识别代码注释

### 5.2 代码展示规范

- 代码块内保留原始缩进
- 关键语句加行内注释说明作用
- 代码前后要有文字说明：这段代码做什么、为什么这样写
- 代码输出/运行结果必须与代码配对展示
