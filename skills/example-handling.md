# 课件例题处理规则

## 核心原则

> **例题是课件的精华。** 授课者花时间讲解的例题，就是考试最可能考察的内容。遗漏例题 = 遗漏考点。

## 1. 例题识别（🔴 不遗漏任何例题）

### 1.1 识别特征（满足任一即判定为例题）

扫描课件文本时，通过以下特征自动识别例题：

| 特征类型 | 具体表现 | 优先级 |
|----------|---------|--------|
| 显式标签 | "例题""例""Example""习题""练习""Exercise""Quiz" | 🔴 最高 |
| 课堂互动题 | 选项A/B/C/D + "提交"按钮 | 🔴 最高 |
| 代码示例 | `# 示例` `# Example` 注释后的完整代码块 | 🟡 高 |
| 问答模式 | 以"？"结尾的段落 + 紧随其后的解答段落 | 🟡 高 |
| 计算过程 | 连续的"已知...求...解..."结构 | 🟡 高 |
| 课件标注 | 课件中标注"重点""必考""掌握"的计算/论述 | 🟡 高 |
| 章节末习题 | 章节结束前的集中题目区域 | 🟢 中 |

### 1.2 例题完整性判定

每道例题提取后，检查是否包含以下要素。缺失的要素必须从上下文推断或标注缺失：

- [ ] 题目陈述（已知条件、所求问题）
- [ ] 题目类型（选择/填空/计算/证明/编程/论述）
- [ ] 解答过程（完整步骤）
- [ ] 最终答案
- [ ] 所属知识点（在哪个概念/公式下讲的）

## 2. 例题解析规范（🔴 禁止只抄步骤不讲解）

### 2.1 理科计算题解析模板

```markdown
✏️ **例题 [编号]**：[题目完整文本]

**📌 考点分析**：
- 本题考察的核心知识点：[列出1-3个]
- 使用的关键公式：[列出公式]
- 考察的能力：[计算能力/概念理解/公式应用/综合分析]

**💡 解题策略选择**：
- 为什么选择这个方法：[解释]
- 有没有其他方法：[如有则列出，说明为什么不选]
- 解题的关键突破口在哪：[指出题目中的关键线索]

**详细解题步骤**：

**步骤1：[步骤名称，如"计算经验熵 H(D)"]**
- 操作：具体的计算过程
- 为什么：这一步基于什么原理/公式
- 中间结果：$$[LaTeX表达式]$$

**步骤2：[步骤名称]**
- 操作：...
- 为什么：...
- 中间结果：$$...$$

**最终答案**：
$$\boxed{[最终答案]}$$

**✅ 验证方法**：
- 如何检查答案正确：[量纲分析/代入验证/边界条件检查/...]

**⚠️ 易错点**：
1. [错误做法]：[为什么错] → [正确做法]
2. [错误做法]：[为什么错] → [正确做法]
```

### 2.2 文科论述题解析模板

```markdown
✏️ **例题 [编号]**：[题目文本]

**📌 考察意图**：
- 本题考察的核心理论/观点
- 期望的回答深度

**答题框架**：
1. [要点1：核心论点]
2. [要点2：论据支撑]
3. [要点3：批判性思考/局限]

**参考答案要点**：[分条列出]

**✅ 评分标准参考**：[如有]
```

### 2.3 代码题解析模板

```markdown
✏️ **例题 [编号]**：[题目描述]

**📌 考察意图**：考察哪个Python库/语法/算法

**代码**：
[完整代码块]

**代码逐行解释**：
- 第X行：[做了什么] → [为什么这样做]
- 第Y行：...

**输出结果**：[代码运行输出]

**⚠️ 易错点**：[常见错误写法及原因]
```

## 3. 举一反三（🔴 每道例题必须附变体题）

### 3.1 变体题生成原则

**核心约束：不能脱离原例题的思维框架。**

| 可以做 | 不可以做 |
|--------|---------|
| 改变数值 | 引入新的知识点 |
| 改变题设场景 | 改变解题方法的核心逻辑 |
| 调整条件组合 | 大幅提高难度 |
| 换一组同类数据 | 引入原题未涉及的公式 |

### 3.2 变体题类型

**类型1：数值变体**（同方法，不同数据）
```
📝 举一反三 1：[新编号]
题目：[原题知识点和方法完全相同，换一组数值]
💡 提示：[简要指出与原题的共同点]
```

**类型2：条件变体**（同方法，条件微调）
```
📝 举一反三 2：[新编号]
题目：[调整题目中的某个条件，但仍用相同方法求解]
💡 提示：[指出需要关注的条件变化]
```

**类型3：逆向变体**（已知结果反推条件）
```
📝 举一反三 3：[新编号]
题目：[告知最终结果，要求反推/验证某个中间量]
💡 提示：[指出逆向思维的关键]
```

### 3.3 变体题数量要求

| 原题类型 | 最少变体题数 |
|----------|-------------|
| 计算题 | 2道（数值变体 + 条件变体） |
| 概念辨析题 | 1道（换一组概念对比） |
| 代码题 | 1道（换数据/场景） |
| 证明题 | 1道（换条件但证法相同） |
| 论述题 | 1道（换角度但框架相同） |

## 4. 例题汇总与索引

### 4.1 章节内例题编号

每章的例题按出现顺序编号：`例ChN-M`（ChN=章号，M=该章内序号）。

### 4.2 例题索引表

在每章末尾生成例题索引表：

```
| 编号 | 题目摘要 | 类型 | 核心考点 |
|------|---------|------|---------|
| 例Ch1-1 | 表达式求值 | 计算 | 运算符优先级 |
| 例Ch1-2 | 参数传递辨析 | 代码分析 | 可变/不可变参数 |
```

## 5. 例题与正文的关系

### 5.1 例题位置

例题必须放在对应知识点讲解**之后**，不能独立于概念存在。

正确的顺序：概念定义 → 公式 → 图解 → **例题** → 举一反三 → 下一概念

### 5.2 例题与概念的链接

每道例题必须在开头标注：
- **所属知识点**：见第X章第Y节
- **前置公式**：需要用到哪些公式（锚点链接到公式速查）
