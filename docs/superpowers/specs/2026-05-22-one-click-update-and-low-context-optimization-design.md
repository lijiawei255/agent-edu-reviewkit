# 一键更新 + 低上下文模型优化 设计文档

日期：2026-05-22
分支：feature/一键更新和针对低上下文模型优化

## 背景

agent-edu-reviewkit 的用户分为两类：仅 clone 使用的路人用户和开发者。路人用户需要便捷的项目更新方式。同时，≤200k 上下文窗口的模型在使用该技能时会面临上下文溢出风险，需要自适应优化。

## 功能一：一键更新脚本

### 设计

- 新建 `update.py`，纯 Python + subprocess，无第三方依赖
- 流程：环境检查 → git fetch → 差异对比 → fast-forward merge → 结果展示
- 异常处理：非 git 仓库、网络失败、不在 main 分支、本地有修改
- 输出为中文，使用 emoji 增强可读性
- 跨平台兼容（Windows/macOS/Linux）

## 功能二：低上下文模型优化

### 设计

- 在 SKILL.md 中新增 Phase 0：用户声明模型和上下文大小
- 根据回答判定模式：🟢 标准模式（>200k）或 🟡 精简模式（≤200k）
- 三项优化策略：
  1. 分批读取材料（Phase 2）：逐章读取而非一次性读取
  2. 主动上下文压缩（全流程检查点）：Phase 2/3/4 完成后触发压缩
  3. 分阶段生成与检查点（Phase 4）：更小批次（1章/脚本）+ 进度检查点
- 1M 上下文模型走标准模式，零影响
