# agent-edu-reviewkit 🎓 理工科课程复习指南生成工具

> ✨ **跨平台 Agent 技能，自动将各类理工科课程材料（PDF/PPTX/DOCX）转换为单文件、自包含的交互式 HTML 复习指南 + 考试押题**。
> 支持数学、物理、电路、计算机、机械、信号处理等所有理工科科目，无需编程知识，跟 AI 对话即可完成。

## 名词解释（快速了解）

如果你是第一次接触 AI 编程工具，以下是一些可能需要了解的名词：

| 名词 | 通俗解释 |
|------|---------|
| **AI 助手 / AI 编程助手** | 一个你可以用自然语言对话的程序，它能帮你运行代码、生成文件。例如 Claude Code、ChatGPT、GitHub Copilot |
| **技能（Skill）** | 一套预先写好的指令，告诉 AI 助手"遇到某种任务时该怎么做"。本项目的技能就是"把课件变成复习文档" |
| **HTML** | 网页文件格式。双击 `.html` 文件就能在浏览器中打开，里面有文字、图片、公式、动画 |
| **Python** | 一种编程语言。你需要安装它来运行本项目的辅助脚本（非常简单的步骤，见下方） |
| **命令行 / 终端** | 一个黑色的文字输入窗口。你需要在这里输入少量命令（如 `pip install`），不用害怕，照着复制粘贴即可 |
| **pip** | Python 的"应用商店"，用于安装 Python 工具包 |
| **git clone** | 从 GitHub 下载项目到你电脑上。推荐克隆到用户主目录（终端默认打开的路径）。如果不会用 git，也可以直接下载 ZIP 压缩包 |
| **CDN** | 网络上的公共文件托管服务。生成的文档需要从 CDN 加载数学公式渲染引擎（也可以离线使用，见 FAQ） |
| **LaTeX** | 数学公式的标准写法。例如 `$E = mc^2$` 会被渲染为漂亮的数学公式 |

## 快速开始（最简单的方式）

### 第 1 步：下载项目

**方式 A（推荐）：从 GitHub 网页下载 ZIP**
1. 打开 https://github.com/lijiawei255/agent-edu-reviewkit
2. 点击绿色的 "<> Code" 按钮 → 选择 "Download ZIP"
3. 解压到你喜欢的目录

**方式 B（如果你会用 git）：**
```bash
git clone https://github.com/lijiawei255/agent-edu-reviewkit.git ~/agent-edu-reviewkit
cd ~/agent-edu-reviewkit
```

> 💡 **推荐克隆到用户主目录（`~`）**，这样打开终端就在项目目录中，无需额外 `cd` 命令。之后只需将课程资料放入 `课程资料/` 文件夹即可使用。

**方式 C（已 clone 过的用户——一键更新到最新版）：**

如果你之前已经用 `git clone` 下载过项目，只需在项目目录下运行：

```bash
python update.py
```

脚本会自动检查远程更新、展示新版本内容并同步到最新。无需重新 clone。**更新不会影响 `课程资料/` 和 `复习文档输出/` 中的内容**，实现无感升级。

### 第 2 步：安装 Python 依赖

打开终端（Windows 上为 PowerShell 或命令提示符），进入项目目录，运行：

```bash
pip install -r requirements.txt
```

这行命令会自动安装所有需要的 Python 包（`pypdf`、`python-pptx`、`python-docx`），均为纯 Python 包，无需任何额外工具。

### 第 3 步：准备课件

将你的课件文件夹放入项目中的 `课程资料/` 目录。每个课程一个子文件夹：

```
课程资料/
├── 数字信号处理/
│   ├── 第1章_绪论.pdf
│   ├── 第2章_离散时间信号与系统.pptx
│   ├── 第3章_DFT.pdf
│   └── exam-scope.json（可选，预设考试范围）
├── 线性代数/
│   ├── chapter1_matrices.pdf
│   └── ...
```

> 💡 **优势**：只需将课件放入 `课程资料/`，技能会自动发现并列出可选课程，无需手动输入路径。所有输出自动保存到 `复习文档输出/<课程名>/` 中，项目结构清晰整洁。

### 第 4 步：告诉 AI 助手你要做什么

**🌐 通用方式（适用于所有 AI 平台，最简单）：**

打开你使用的 AI 助手（Claude Code、ChatGPT、Kimi Chat 等），发送：

> 请按照 skills/course-review-guide/SKILL.md 中的指引，帮我把 [课件目录路径] 中的课件转化为 HTML 考试复习文档。

如果你使用的是聊天式 AI（非编程助手），需要同时把项目中 `skills/course-review-guide/SKILL.md` 的内容复制粘贴到对话中。

**🤖 Claude Code 用户（推荐）：**

在项目目录中启动 Claude Code，直接说：

```
请帮我整理课程复习文档
```

Claude Code 会自动加载技能。也可以用命令显式触发：`/skill:course-review-guide`

**🔧 OpenCode 用户：**

```
@course-review 帮我生成复习文档，课件在 [目录路径]
```

### 第 5 步：跟随 AI 的指引

AI 助手会引导你逐步完成：
1. 确认考试范围（考哪些章节）
2. 运行提取脚本 `extract_course_materials.py`（AI 会告诉你准确的命令）
3. 运行智能图片匹配 `match_images.py --interactive` 生成图片-章节映射
4. 生成交互式 HTML 复习文档（基于 image_mapping.json 智能嵌入图片）
5. 运行 `python embed_images.py <文件名>.html --in-place -v` 把图片内嵌到 HTML 中
6. （可选）运行 `python setup_mathjax.py` 下载数学公式引擎，实现离线查看
7. （可选）生成考试押题文档

最终得到一个 `.html` 文件，双击用浏览器打开即可开始学习。

## 详细使用说明

### 工作流程概览

整个流程分为七个阶段，AI 助手会按顺序引导你：

```
课件文件 ──→  P1: 范围确认  ──→  P2: 内容提取  ──→  P2+: 图片智能匹配  ──→  P3: 补充调研
                                                                                        │
                                                                                        ▼
                          HTML复习文档  ←──  P5: 质量检查(36项)  ←──  P4: HTML生成  ←──┘
                                                                                        │
                                                                                        ▼
                                                                          P6: 考试押题（可选）
```

### Phase 1：范围确认

AI 助手会**自动扫描 `课程资料/` 目录**，列出检测到的课程文件夹并请你选择。选定后自动设置好课件路径和输出路径。

确认内容包括：

- 考试范围（哪些章节考、哪些不考）
- 课程名称、授课教师、使用教材
- 考试形式（闭卷 / 开卷）
- 输出文件名（默认保存到 `复习文档输出/<课程名>/`）

**建议**：在课程文件夹中放入 `exam-scope.json` 预设考试范围，可跳过手动确认。也可直接告诉 AI 你的要求。

### Phase 2：内容提取

AI 助手会引导你运行 `extract_course_materials.py` 脚本。**推荐使用命令行参数**（无需编辑代码）：

```bash
# 基础用法：指定课件目录和输出目录
python extract_course_materials.py --course-dir "课程资料/你的课程名" --output-dir "复习文档输出/你的课程名"

# 常用完整命令示例：
python extract_course_materials.py \
    --course-dir "课程资料/数字信号处理" \
    --output-dir "复习文档输出/数字信号处理"
```

提取结果会自动保存到 `复习文档输出/<课程名>/extracted_text/` 和 `复习文档输出/<课程名>/extracted_images/`。

**可选的 --render-pages 参数**：如果你的 AI 助手支持"看图"（多模态模型，如 Claude Opus 4、GPT-4V），强烈建议添加此参数：

```bash
# 需要先安装 pymupdf: pip install pymupdf
python extract_course_materials.py --course-dir "课程资料/你的课程名" --render-pages
```

这个参数会将 PDF 页面渲染为整页截图，让 AI 能看到课件的完整排版（公式、图表、流程图的位置关系），**大幅提升生成质量**。扫描版 PDF（纯图片课件）尤其需要此参数。

脚本会自动完成：
- 提取所有课件的文本和图片
- 检测纯图片型课件（扫描版 PDF 等）并单独标注
- 在文本中用 `[图片: xxx.png]` 标记图片对应的位置
- 输出详细的提取报告

### Phase 2+：图片智能匹配（新增）

提取完成后，运行 `match_images.py` 进行图片-章节智能匹配：

```bash
# 基础用法
python match_images.py --text-dir "复习文档输出/<课程名>/extracted_text" --image-dir "复习文档输出/<课程名>/extracted_images"

# 交互式确认模式（推荐，可手动修正匹配结果）
python match_images.py --text-dir "复习文档输出/<课程名>/extracted_text" --image-dir "复习文档输出/<课程名>/extracted_images" --interactive

# 指定输出映射文件
python match_images.py --text-dir "复习文档输出/<课程名>/extracted_text" -o image_mapping.json
```

功能包括：
- 解析所有 `[图片: xxx.png]` 标记，提取前后各3段上下文
- 基于关键词自动识别图片类型（6大类：示意图、波形谱图、公式推导图、例题配图、对比分析图、物理模型图）
- 从上下文自动推断图片所属章节
- 生成置信度评分，优先嵌入高置信度图片
- 交互式确认模式支持用户手动修正章节归属
- 输出 `image_mapping.json` 供后续 HTML 生成使用

### Phase 3：补充调研（可选）

AI 助手可能会搜索以下来源来增强对课程内容的理解：
- GitHub 上的开源课程笔记和习题解答
- Wikipedia 的概念定义

**所有外部内容仅供参考**，最终以你提供的课件为准。

### Phase 4：HTML 生成

AI 助手基于全部课件内容和 `image_mapping.json` 生成一个**交互式** HTML 复习文档。**文档规模巨大（2000+行），采用串行 Python 脚本分批追加模式生成**——先用 Write 工具写入 CSS 和页面结构，再用 Python 脚本逐批追加章节内容（每批2-3章），最后追加附录和 JavaScript。避免一次性生成导致超时或输出截断。

生成的文档包含：
- **MathJax** 渲染数学公式（LaTeX 格式）
- **课件原图智能嵌入**（基于 image_mapping.json，优先嵌入高置信度匹配的课件图片，配有自动生成的图注）
- **补充 SVG 示意图**（仅在课件无对应图片时作为补充，使用 `<figure>` + `<img>` 规范标签）
- **响应式 CSS**（电脑、平板、手机都能看 + 打印样式 + 宽屏侧边目录）
- **交互式组件**（可折叠推导、选项卡、闪卡、测验、进度追踪、搜索）
- **内联 JavaScript**（无需任何外部 JS 库，单文件自包含）

生成完成后，运行以下命令使 HTML 完全自包含：

```bash
# 图片内嵌为 base64（质量检测 + 去重 + 详细统计）
python embed_images.py "复习文档输出/<课程名>/<课程名>_复习指南.html" --verbose

# 原地替换模式
python embed_images.py "复习文档输出/<课程名>/<课程名>_复习指南.html" --in-place -v

# 仅分析图片目录质量，不生成 HTML
python embed_images.py --analyze-only

python setup_mathjax.py               # （一次性）下载 MathJax 到本地，实现离线查看公式
```

`embed_images.py` 增强功能：
- 图片魔数校验，损坏文件自动跳过
- 4级质量评级：excellent (>100KB) / good (>30KB) / poor (>10KB) / too_small
- MD5 哈希自动去重，避免同一图片多次嵌入
- 缺失图片和低质量图片详细报告
- 支持递归目录查找同名图片文件
- 支持 srcset 响应式图片转换

### Phase 5：质量检查

AI 助手自动进行 **36 项质量检查**，涵盖结构完整性、交互功能、数学排版、图片嵌入、打印兼容等方面，确保输出完整可用。

新增图片与闪卡专项检查：
- ✅ 每章图片数量达标（8-10张）
- ✅ 核心概念均有对应配图
- ✅ 图片均配有上下文相关的图注说明
- ✅ 图片路径正确，可被 embed_images.py 找到
- ✅ 闪卡无双击翻转 bug，事件绑定正确
- ✅ 闪卡 CSS 3D 翻转效果正常

### Phase 6：考试押题（可选）

完成复习文档后，AI 助手可基于课程内容分析生成一份**独立的 HTML 考试押题文档**——包含高频考点预测、分章节练习题（选择题、填空题、计算题）、模拟试卷及答案解析。押题文档使用试卷风格排版（衬线字体、极简布局），独立于复习文档，便于单独打印和分发。

### 图片内容识别与交叉验证

采用**三级视觉降级 + 交叉验证**策略，确保图片内容准确理解：

| 优先级 | 方案 | 说明 |
|--------|------|------|
| 1 | AI 模型自带视觉 | 直接读取图片内容，生成详细内容描述（需要支持多模态的模型） |
| 2 | MCP 视觉服务器 | 通过 MCP 协议调用外部视觉工具进行图片分析 |
| 3 | 上下文关键字匹配 | 提取图片前后3段文本，通过关键词进行分类和章节归属判断 |

**交叉验证机制**：
- 上下文关键词分类 → 图片视觉内容描述 → 两者一致才标记为高置信度
- 高置信度图片优先嵌入章节正文中
- 低置信度图片放入附录供用户手动调整
- 图片说明文字自动结合上下文和视觉描述生成

## 交互式学习功能

生成的 HTML 复习文档包含以下交互式学习组件（参考自 [Thariq 的《Using Claude Code: The Unreasonable Effectiveness of HTML》](https://x.com/thariqk)）：

### 可折叠推导步骤

所有数学推导默认折叠，点击展开。节省页面空间，适合按需查看。打印时自动展开所有内容。

### 选项卡视图

每章提供两个选项卡：
- **⚡ 快速复习**：章节概要、术语闪卡、关键公式表格
- **📖 详细讲解**：完整推导、例题详解、配图说明

### 术语闪卡（Flashcard）

核心术语的翻转卡片——正面显示术语名称，点击翻转显示定义或公式。CSS 3D 翻转效果，支持鼠标和触控。

**闪卡防双击保护机制**（已修复双击不翻转 bug）：
- CSS 层：`pointer-events: none` 禁止内部元素触发点击
- JS 层：`event.stopPropagation()` 阻止事件冒泡
- 防抖层：300ms 冷却期，防止短时间内重复点击
- 事件委托：使用事件委托而非内联 onclick，避免重复绑定

### 练习测验

每章末有 2-4 道概念检查题，答案默认隐藏，点击"显示答案"查看解析。适合自我检测。

### 学习进度追踪

侧边栏目录包含勾选框，勾选已完成的章节，顶部进度条实时更新。进度保存在浏览器 localStorage 中，刷新不丢失。

### 全文搜索

文档顶部搜索栏支持按关键词过滤章节，快速定位需要的概念或公式。

### 打印兼容

打印时所有交互元素自动降级：折叠内容展开、所有选项卡显示、闪卡展开为正面+背面、搜索和进度控件隐藏。

## 自主智能体支持

OpenClaw、Hermes 等自主智能体全自动运行，**不保证回合制用户交互**。本技能针对自主模式做了专门适配。

### exam-scope.json 配置文件

在课件目录中放置 `exam-scope.json`，自主 agent 会自动读取并跳过交互式范围确认：

```json
{
  "course_name_zh": "数字信号处理",
  "course_name_en": "Digital Signal Processing",
  "instructor": "张三",
  "textbook": "《数字信号处理》，程佩青，第四版",
  "exam_type": "闭卷",
  "language": "zh",
  "chapters_in_scope": ["第1章", "第2章", "第3章", "第4章", "第5章"],
  "chapters_excluded": ["第6章"],
  "output_filename": "DSP_期末复习指南.html"
}
```

模板见项目根目录的 `exam-scope-template.json`。

### 三级范围确定策略

| 级别 | 方法 | 需要交互 | 可靠性 |
|------|------|----------|--------|
| 1 | 读取 `exam-scope.json` | 不需要 | 高 |
| 2 | 从文件名和内容推断范围 | 不需要 | 中 |
| 3 | 通过消息通道请求 | 需要（异步） | 高 |

### 自动推断模式

如果 `exam-scope.json` 不存在，自主 agent 会从课件文件名推断章节范围，生成的 HTML 顶部会显示醒目的自主模式横幅，提醒核对考试范围。同时会在课件目录自动生成 `exam-scope-template.json`（含推断默认值）供下次使用。

详见 `AGENTS.md`。

## 隐私与安全提示

使用本工具时，你的课件内容会经过以下环节：

1. **本地提取**：`extract_course_materials.py` 在你的电脑上运行，提取的文字和图片保存在本地，**不会上传**到任何服务器
2. **AI 服务商**：你在 AI 助手中对话时，课件内容会被发送到你使用的 AI 服务商（如 Anthropic、OpenAI、Moonshot 等），请参考对应服务商的隐私政策
3. **网络搜索**：AI 助手可能会访问 GitHub、Wikipedia 等公共网站进行补充搜索

**建议**：
- 如果课件包含个人敏感信息（如学号、身份证号），请先移除再处理
- 如果课件包含未发表的研究成果或商业机密，请先确认 AI 服务商的数据使用政策
- 所有提取的中间文件（`extracted_text/`、`extracted_images/`）保存在你的电脑本地，你可以随时删除

## 常见问题

### Q: 我的课件是扫描版 PDF（纯图片），没有任何文字，怎么办？

如果扫描质量较好，支持多模态的 AI 模型可以直接识别图片中的内容。建议：
1. 使用 `--render-pages` 参数运行提取脚本（需先 `pip install pymupdf`），将页面渲染为清晰截图
2. 使用支持视觉的模型（Claude Opus 4、GPT-4V 等）
3. 如果以上不可行，先用 OCR 工具（如 Tesseract：`pip install pytesseract pdf2image`）将图片转为文字

### Q: 公式提取后不完整或显示为乱码？

PDF 文本提取可能导致公式损坏。本技能会自动检测并修复 6 类常见公式问题（断裂、符号丢失、分数线丢失、上下标错位、矩阵混乱、函数名错误）。所有公式均以 LaTeX 格式重新排版，确保在 HTML 中正确渲染。

### Q: 生成的 HTML 文件太大怎么办？

CSS 和内联 SVG 占用空间很小。运行 `embed_images.py` 将图片内嵌为 base64 后，文件会增大（约为原始图片总大小的 1.37 倍），但换来的是完全自包含、可单文件分享的便利。如果文件太大，可以不运行内嵌脚本，改为把 HTML 与 `extracted_images/` 文件夹一起打包分享。

### Q: 如何分享和打印复习文档？

- **分享**：运行 `python embed_images.py <文件名>.html` 得到 `_embedded.html` 文件（图片全部内嵌），发送这一个文件即可
- **打印**：在浏览器中打开 HTML，按 `Ctrl+P`（或 `Cmd+P`）即可。内置的 `@media print` 样式会自动优化排版，所有折叠内容自动展开

### Q: 交互功能（闪卡、测验等）需要联网吗？

不需要。所有交互功能使用纯 HTML/CSS/JavaScript 实现，无任何外部 JS 依赖。唯一需要联网的是数学公式渲染。如果需要在完全无网络环境下查看公式，请运行 `python setup_mathjax.py`（一次性）下载 MathJax 到本地。

### Q: 如何在无网络环境下查看公式？

**最简单的方式**：运行项目自带的一键安装脚本：

```bash
python setup_mathjax.py
```

这会自动下载 MathJax 3 公式渲染引擎到项目的 `mathjax/` 目录。生成的 HTML 会**自动优先使用本地 MathJax**，加载失败时才从网络加载。无网络时公式依然正常渲染。

如果需要把 HTML 文件分享给别人：将文件连同 `mathjax/` 文件夹一起打包发送即可。

### Q: 学习进度会丢失吗？

进度保存在浏览器的 localStorage 中，刷新页面不会丢失。注意：进度与文件路径绑定，如果移动文件位置，进度会重置。

### Q: 支持哪些课件格式？

| 格式 | 扩展名 | 支持状态 |
|------|--------|---------|
| PDF | `.pdf` | ✅ 完全支持 |
| PowerPoint | `.pptx`, `.pptm` | ✅ 完全支持 |
| Word | `.docx`, `.dotx`, `.dotm` | ✅ 完全支持 |
| 旧版 PowerPoint | `.ppt` | ❌ 不支持，请用 PowerPoint 另存为 `.pptx` |
| 旧版 Word | `.doc` | ❌ 不支持，请用 Word 另存为 `.docx` |

### Q: 我的课程是英文授课，能用吗？

可以。本技能支持中英双语输出——关键术语会同时给出中英文对照。如果全英文授课，请在与 AI 助手确认范围时说明。

### Q: 如何用 OpenClaw/Hermes 自主生成复习文档？

1. 在课件目录中创建 `exam-scope.json`（参考 `exam-scope-template.json` 填写）
2. 将项目克隆到 OpenClaw 可访问的位置
3. OpenClaw agent 会自动读取 `AGENTS.md` 中的指令，执行所有 Phase
4. 生成完成后检查文档顶部的自主模式横幅，确认考试范围正确
